El modal hace re-dispatch: recibe de sus hijos y reemite al padre (3 niveles).

3 · Secuencia end-to-end

1
El comprador hace clic en "Comprar"
En una card de la PLP.
lvlupQuickBuyButton → buyclick { productId }
2
El padre abre el modal y pide datos al servidor
Estado loadingProduct. Llamada Apex con el productId.
@AuraEnabled getProductPurchaseInfo(productId) → ProductPurchaseInfo
3
Apex consulta el producto + precio
SOQL sobre Product2 (campos custom de inventario) y PricebookEntry. Clase without sharing.
4
El modal muestra el producto
Imagen, SKU, stock, cantidad = MOQ por defecto.
5
El comprador ajusta cantidad / opciones
Los hijos validan su parcela y avisan al padre (vía el modal).
quantitychange · stockvalidated · attributechange
6
El padre recalcula la validez global
recompute(): MOQ, múltiplo, stock, atributos → uiState. "Añadir" se habilita solo si todo es válido.
7
Confirmar → add-to-cart real
Estado addingToCart. API estándar del carrito.
commerce/cartApi → addItemToCart(productId, quantity)
8
Feedback y carrito actualizado
"Añadido ✓", el contador del carrito sube. addedSuccess / addError.
